# 사주 던전 크롤러 — 설계 문서

> 사주명리의 미궁을 카드 기반 던전 크롤러로 확장하는 설계

## 1. 개요

### 목적
현재 MUD 스타일 사주풀이 앱을 **카드 공개형 스토리텔링 + 덱 빌딩 던전 크롤러**로 전환한다. 전투는 사주 풀이를 전달하는 연출 수단이며, 핵심은 여전히 AI 기반 사주 해석이다.

### 타겟 사용자
사주에 관심 있는 MZ세대 (20-30대). 가벼운 운세/성격 테스트를 좋아하고, 게임적 재미와 반복 방문 동기를 원하는 사용자.

### 핵심 원칙
- **사주 풀이가 주인공** — 전투는 풀이 전달을 위한 연출
- **내 사주의 위험 요소가 적** — 기신, 흉살, 상극 오행이 몬스터로 등장
- **CRT 터미널 미학 유지** — ASCII 카드 아트, 기존 레트로 분위기 보존
- **서버리스** — 모든 게임 상태 localStorage, DB 불필요

## 2. 게임 구조

### 2.1 메인 플레이 (1회성 딥 플레이)

```
[사주 입력] → [카드 공개 세레모니] → [던전 탐험 5층] → [엔딩 리포트]
```

**카드 공개 세레모니:** 사주 계산 후, 8장의 카드(4주 × 천간/지지)가 한 장씩 뒤집히며 공개. 각 카드 공개 시 AI 짧은 내레이션(2-3문장).

**던전 탐험:** 5개 층을 순서대로 진행. 각 층에서 위험 요소(적) 조우 → 카드 선택(대응) → AI 풀이.

**엔딩 리포트:** 모든 층 클리어 후 종합 성적표 — 획득 카드, 전투 기록, AI 종합 풀이.

### 2.2 일일 콘텐츠 (리텐션)

- **오늘의 시련:** 당일 일진(日辰) 기반 몬스터, 내 사주와의 관계로 난이도 결정
- **일일 운세:** 전투 후 AI가 오늘의 운세 풀이
- **일일 보상 카드:** 60간지 컬렉션용 카드 1장

### 2.3 리텐션 시스템

| 시스템 | 내용 |
|--------|------|
| 방문 스트릭 | 연속 방문 일수, 7일마다 특수 카드 |
| 컬렉션 | 60간지(갑자~계해) 카드 수집 |
| 레벨 | 전투 횟수 기반 칭호 (초심자→수련생→도사→선인) |
| 업적 | "모든 오행 카드 수집", "10일 연속 방문" 등 |

## 3. 던전 층 구조

```
╭──────────────────────────────────╮
│         사주명리의 미궁           │
│                                  │
│  [1층] 운명의 입구 — 카드 공개    │
│          ↓                       │
│  [2층] 종합의 방 — 종합 운세      │
│          ↓                       │
│  [3층] 오행의 시련 — 상세 분석    │
│          ↓                       │
│  [4층] 시간의 회랑 — 대운/세운    │
│          ↓                       │
│  [5층] 심연의 거울 — 궁합/공망    │
│                                  │
│  [일일] 오늘의 시련 — 일일 운세   │
╰──────────────────────────────────╯
```

### 층별 상세

| 층 | 테마 | 적 생성 원리 | AI 풀이 주제 |
|----|------|------------|-------------|
| 1층 | 카드 공개 세레모니 | 적 없음 | 사주 원국 소개, 일간 성격 |
| 2층 | 종합의 방 | 기신(忌神) 몬스터 | 종합 운세, 성격, 적성 |
| 3층 | 오행의 시련 | 과다/부족 오행이 적 | 오행 밸런스, 건강, 재물 |
| 4층 | 시간의 회랑 | 대운/세운 충(沖)·형(刑) | 시기별 운세, 주의 시기 |
| 5층 | 심연의 거울 | 공망 + 흉살 최종보스 | 깊은 자아 분석, 궁합 |

### 각 층의 흐름

```
[층 진입 → 분위기 묘사 (현재 room description과 동일)]
    ↓
[위험 요소(적) 등장 + AI가 이 요소가 내 사주에서 무엇을 의미하는지 설명]
    ↓
[카드 선택 — 3장 중 1장 선택 (간단한 대응)]
    ↓
[오행 상생/상극 판정 → 승/패]
    ↓
[본격 AI 사주 풀이 (현재와 동일한 깊이의 스트리밍 풀이)]
  - 승리 → 긍정적 관점에서 풀이
  - 패배 → 주의점/조언 관점에서 풀이
  (풀이 내용 자체는 동일, 톤만 다름)
    ↓
[보상 카드 획득 + 다음 층으로]
```

## 4. 카드 시스템

### 4.1 카드 비주얼 (ASCII)

```
╭─────────────╮
│ 壬 (임)     │
│   ━━━━━     │
│    水水      │
│   ～～～    │
│             │
│  양(+) 수   │
│  ATK:15     │
│  일간(日干) │
╰─────────────╯
```

각 카드에 표시: 한자, 한글 읽기, 오행 심볼, 음양, ATK/DEF, 역할(일간/월간 등)

### 4.2 플레이어 덱 (사주에서 자동 생성)

**기본 카드 (8장):** 4주의 천간 4장 + 지지 4장. 사주 계산 결과에서 자동 생성.

| 카드 출처 | 수량 | 예시 |
|----------|------|------|
| 년주 천간/지지 | 2장 | 庚(경금), 午(오화) |
| 월주 천간/지지 | 2장 | 丁(정화), 亥(해수) |
| 일주 천간/지지 | 2장 | 壬(임수), 辰(진토) |
| 시주 천간/지지 | 2장 | 甲(갑목), 寅(인목) |

**보너스 카드 (사주 분석에서):**
- 용신(用神) 카드: 특수 강화 능력 보유
- 십성 카드: 비견/식신/편재 등 고유 효과
- 길신살 카드: 방어/회복 효과

**수치 산정:**
- ATK: 오행 기본값(10) + 음양 보정(양+5) + 역할 보정(일간+10, 용신+15)
- DEF: 12운성 기반 (제왕=20, 장생=15, 사=5)

### 4.3 몬스터 (내 사주의 위험 요소)

| 적 유형 | 생성 원리 | 예시 |
|---------|----------|------|
| 기신(忌神) 몬스터 | 일간을 해치는 오행 | 수 일간 + 토 기신 → "토의 장벽" |
| 상극 오행 보스 | 과다/부족 오행 실체화 | 화 과다 → "폭주하는 화염" |
| 흉살(凶殺) 몬스터 | 나쁜 신살이 실체화 | 도화살 → "유혹의 꽃요괴", 겁살 → "그림자 도적" |
| 공망(空亡) 보스 | 공망 지지가 최종 보스 | "허무의 심연" |
| 세운 이벤트 몬스터 | 올해 세운과 충/형 | 2026 병오 + 자수 → "오자충 폭풍" |

**몬스터 수치:** HP(50-100), ATK(10-30), 오행 속성, 특수 능력(억제/강화)

### 4.4 전투 메카닉스

```
[몬스터 등장 + AI 내레이션 (이 요소가 내 사주에서 의미하는 것)]
    ↓
[내 덱에서 3장 랜덤 드로 → 1장 선택]
    ↓
[오행 상생/상극 판정]
  - 상극(내가 이기는 관계) → 2배 데미지
  - 상생(내가 살려주는 관계) → 회복/버프
  - 같은 오행(비겁) → 동맹 효과 (1.5배)
  - 역상극(내가 지는 관계) → 0.5배 데미지
    ↓
[결과 판정 → 승리/패배]
    ↓
[AI 스트리밍 사주 풀이 시작 (승패에 따라 톤 변화)]
```

**전투 터미널 UI 예시:**
```
╔═══════════════════════════════╗
║  ⚔ 기신(忌神) 등장!           ║
║  "토의 장벽" (戊土)           ║
║  HP: ████████░░ 80/100       ║
║  속성: 토(土) | ATK: 25       ║
║  특성: 수기운(水) 억제         ║
╚═══════════════════════════════╝

현자: "그대의 수 기운을 막아서는
토의 장벽이로다..."

─── 카드를 선택하시오 ───
[1] 甲木 (갑목) ATK:20 → 상극! 효과적
[2] 壬水 (임수) ATK:15 → 억제당함
[3] 庚金 (경금) ATK:18 → 토생금 상생
```

## 5. 일일 콘텐츠 상세

### 오늘의 시련

- **생성:** 매일 자정, 당일 일진(日辰)의 천간+지지로 몬스터 생성
- **난이도:** 내 사주와의 관계로 결정
  - 일진이 내 용신과 같은 오행 → 쉬움 (HP 낮음)
  - 일진이 내 기신과 같은 오행 → 어려움 (HP 높음)
  - 일진이 내 일간과 충(沖) → 특수 이벤트
- **풀이:** 전투 후 AI가 오늘의 운세를 사주 관점에서 풀이
- **보상:** 해당 일진의 간지 카드 (60간지 컬렉션)

### 저장 구조 (localStorage)

```typescript
interface GameSave {
  // 기본 정보
  birthInfo: BirthInfo;
  sajuResult: SajuResult;

  // 메인 플레이 진행
  currentFloor: number;
  deck: Card[];
  completedFloors: number[];
  battleHistory: BattleRecord[];
  aiCache: Record<string, string>;

  // 리텐션
  level: number;
  streak: number;
  lastVisit: string; // ISO date
  collection: string[]; // 수집한 간지 카드 ID
  achievements: string[];
  dailyCompleted: string; // 오늘 일일 완료 여부 (ISO date)
}
```

### 6.3 상태머신 확장

현재 phase:
```
intro → name → date → calendar → time → gender → marriage → exploring
```

확장 후:
```
intro → name → date → calendar → time → gender → marriage
  → card_reveal (새로운: 카드 공개 세레모니)
  → exploring (층 탐험)
  → battle (전투 중)
  → reward (보상 획득)
  → daily (일일 시련)
```

### 6.4 AI 프롬프트 확장

기존 `getRoomPrompt()` 확장:
- 전투 결과(승/패) 컨텍스트 추가
- 몬스터 정보(어떤 위험 요소인지) 전달
- 승리 시: "이 위험 요소를 극복한 관점에서" 풀이
- 패배 시: "이 위험 요소에 주의해야 하는 관점에서" 풀이

## 7. 검증 방법

1. **카드 생성 검증:** 사주 입력 → 8장 기본카드가 올바른 오행/수치로 생성되는지 확인
2. **몬스터 생성 검증:** 기신/흉살이 올바르게 몬스터로 변환되는지 확인
3. **전투 판정 검증:** 상생/상극 데미지 계산이 오행 원리에 맞는지 확인
4. **AI 풀이 검증:** 승/패에 따라 풀이 톤이 달라지는지 확인
5. **일일 콘텐츠 검증:** 날짜별 다른 몬스터 생성, localStorage 저장/복원
6. **E2E 흐름:** 사주 입력 → 카드 공개 → 5층 클리어 → 엔딩 리포트 전체 흐름
7. **리텐션 검증:** 재방문 시 저장된 데이터 로드, 스트릭 계산

## 8. 구현 우선순위 (제안)

1. **Phase 1:** 카드 시스템 + 카드 공개 세레모니 (기존 사주 계산 → 카드 변환)
2. **Phase 2:** 몬스터 생성 + 전투 메카닉스 (기신/흉살 → 몬스터, 상생/상극 판정)
3. **Phase 3:** 던전 층 구조 (rooms → floors 전환, 층별 전투 통합)
4. **Phase 4:** AI 프롬프트 확장 (전투 결과 컨텍스트)
5. **Phase 5:** 일일 콘텐츠 + 리텐션 (일일 운세, 스트릭, 컬렉션)
6. **Phase 6:** 저장/불러오기 + 폴리싱
