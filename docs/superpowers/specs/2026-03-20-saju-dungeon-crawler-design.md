# 사주 던전 크롤러 — 설계 문서

> 사주명리의 미궁을 카드 기반 던전 크롤러로 확장하는 설계

## 1. 개요

### 목적
현재 MUD 스타일 사주풀이 앱을 **카드 공개형 스토리텔링 + 덱 빌딩 던전 크롤러**로 전환한다. 전투는 사주 풀이를 전달하는 연출 수단이며, 핵심은 여전히 AI 기반 사주 해석이다.

### 타겟 사용자
사주에 관심 있는 MZ세대 (20-30대). 가벼운 운세/성격 테스트를 좋아하고, 게임적 재미와 반복 방문 동기를 원하는 사용자.

### 핵심 원칙
- **사주 풀이가 주인공** — 전투는 풀이 전달을 위한 연출
- **내 사주의 위험 요소가 적** — 기신, 흉살, 상극 오행이 몬스터로 등장
- **CRT 터미널 미학 유지** — ASCII 카드 아트, 기존 레트로 분위기 보존
- **서버리스** — 모든 게임 상태 localStorage, DB 불필요

## 2. 게임 구조

### 2.1 메인 플레이 (1회성 딥 플레이)

```
[사주 입력] → [카드 공개 세레모니] → [던전 탐험 4층] → [엔딩 리포트]
```

**카드 공개 세레모니:** 사주 계산 후, 8장의 카드(4주 × 천간/지지)가 한 장씩 뒤집히며 공개. 각 카드 공개 시 AI 짧은 내레이션(2-3문장). 전투 없이 카드 공개 + 내레이션만 진행.

**던전 탐험:** 4개 전투 층을 순서대로 진행. 각 층에서 위험 요소(적) 조우 → 카드 선택(대응) → AI 풀이.

**엔딩 리포트:** 모든 층 클리어 후 종합 리포트 표시.

### 2.2 엔딩 리포트 상세

모든 층 클리어 후 표시되는 최종 화면:

```
══════════════════════════════════
  ⚔ 던전 클리어! — {이름}의 사주 여정
══════════════════════════════════

▸ 전투 기록
  2층 종합의 방: 승리 (甲木 → 토의 장벽)
  3층 오행 시련: 패배 (壬水 → 폭주하는 화염)
  4층 시간 회랑: 승리 (庚金 → 오자충 폭풍)
  5층 심연 거울: 승리 (丁火 → 허무의 심연)

▸ 획득 카드: 12장
  기본 8장 + 보상 4장

▸ 종합 평가
  [AI 종합 풀이 — 전투 결과를 반영한 최종 해석]

──────────────────────────────────
[다시 시작] [일일 시련] [내보내기]
```

- AI 종합 풀이: 전체 전투 결과(승/패 패턴)를 프롬프트에 포함하여 최종 종합 해석 요청
- 내보내기: 현재와 동일한 `.txt` 내보내기 + 전투 기록 포함

### 2.3 일일 콘텐츠 (리텐션)

- **오늘의 시련:** 당일 일진(日辰) 기반 몬스터, 내 사주와의 관계로 난이도 결정
- **일일 운세:** 전투 후 AI가 오늘의 운세 풀이
- **일일 보상 카드:** 60간지 컬렉션용 카드 1장

### 2.4 리텐션 시스템

| 시스템 | 내용 |
|--------|------|
| 방문 스트릭 | 연속 방문 일수, 7일마다 특수 카드 |
| 컬렉션 | 60간지(갑자~계해) 카드 수집 |
| 레벨 | 전투 횟수 기반 칭호: 초심자(0)→수련생(5)→도사(15)→선인(30) |
| 업적 | "오행 완성(5종 수집)", "10일 연속", "전층 승리", "전층 패배 후 재도전 승리" |

## 3. 던전 층 구조

```
╭──────────────────────────────────╮
│         사주명리의 미궁           │
│                                  │
│  [1층] 운명의 입구 — 카드 공개    │
│          ↓                       │
│  [2층] 종합의 방 — 종합 운세      │
│          ↓                       │
│  [3층] 오행의 시련 — 상세 분석    │
│          ↓                       │
│  [4층] 시간의 회랑 — 대운/세운    │
│          ↓                       │
│  [5층] 심연의 거울 — 궁합/공망    │
│                                  │
│  [일일] 오늘의 시련 — 일일 운세   │
╰──────────────────────────────────╯
```

### 층별 상세

| 층 | 테마 | 흐름 유형 | 적 생성 원리 | AI 풀이 주제 |
|----|------|----------|------------|-------------|
| 1층 | 카드 공개 세레모니 | **비전투** (카드 공개만) | 적 없음 | 사주 원국 소개, 일간 성격 |
| 2층 | 종합의 방 | 전투 | 기신(忌神) 몬스터 | 종합 운세, 성격, 적성 |
| 3층 | 오행의 시련 | 전투 | 과다/부족 오행이 적 | 오행 밸런스, 건강, 재물 |
| 4층 | 시간의 회랑 | 전투 | 대운/세운 충(沖)·형(刑) | 시기별 운세, 주의 시기 |
| 5층 | 심연의 거울 | 전투 | 공망 + 흉살 최종보스 | 깊은 자아 분석, 궁합 |

### 1층 흐름 (비전투 — 카드 공개)

```
[1층 진입 → "운명의 입구" 분위기 묘사]
    ↓
[사주표 표시 (현재 PillarDisplay와 동일)]
    ↓
[8장 카드 순차 공개 — 각 카드마다 짧은 AI 내레이션]
  년주 천간 → 년주 지지 → 월주 천간 → ... → 시주 지지
    ↓
[덱 완성 안내 + "다음 층으로 (Enter)"]
```

### 2~5층 흐름 (전투)

```
[층 진입 → 분위기 묘사]
    ↓
[위험 요소(적) 등장 + AI가 이 요소가 내 사주에서 무엇을 의미하는지 설명]
    ↓
[카드 선택 — 3장 중 1장 (간단한 1턴 대응)]
    ↓
[오행 상생/상극 판정 → 승/패]
    ↓
[본격 AI 사주 풀이 (현재와 동일한 깊이의 스트리밍 풀이)]
  - 승리 → 긍정적 관점에서 풀이
  - 패배 → 주의점/조언 관점에서 풀이
    ↓
[보상 카드 획득 + "다음 층으로 (Enter)" / 클리어 후 완료 층 재방문 가능]
```

### 층 탐색 규칙

- **진행:** 선형 (1→2→3→4→5), 첫 플레이 시 순서대로만 진행 가능
- **재방문:** 클리어한 층은 자유롭게 재방문 가능 (AI 풀이 캐시에서 표시, 전투 재도전 선택 가능)
- **내비게이션:** 층 클리어 후 "이동" 명령으로 이전 층 선택 가능 (현재 방 번호 선택과 동일)

## 4. 카드 시스템

### 4.1 카드 비주얼 (ASCII)

**데스크톱 (15자 폭):**
```
╭─────────────╮
│ 壬 (임)     │
│   ━━━━━     │
│    水水      │
│   ～～～    │
│             │
│  양(+) 수   │
│  ATK:15     │
│  일간(日干) │
╰─────────────╯
```

**모바일 (9자 폭, 카드 3장 나란히 = 27자 + 여백):**
```
╭───────╮
│壬(임) │
│ 水水  │
│양 수  │
│ATK:15 │
╰───────╯
```

각 카드에 표시: 한자, 한글 읽기, 오행 심볼, 음양, ATK, 역할

### 4.2 플레이어 덱 (사주에서 자동 생성)

**기본 카드 (8장):** 4주의 천간 4장 + 지지 4장. 사주 계산 결과에서 자동 생성.

| 카드 출처 | 수량 | 예시 |
|----------|------|------|
| 년주 천간/지지 | 2장 | 庚(경금), 午(오화) |
| 월주 천간/지지 | 2장 | 丁(정화), 亥(해수) |
| 일주 천간/지지 | 2장 | 壬(임수), 辰(진토) |
| 시주 천간/지지 | 2장 | 甲(갑목), 寅(인목) |

**보상 카드 (전투 승리 시):**
- 각 층 전투 승리 시 1장, 해당 몬스터의 약점 오행 카드

### 4.3 카드 수치 산정

**ATK 계산:**
```
기본값 = 10
+ 양(陽)이면 +5, 음(陰)이면 +0
+ 천간(天干)이면 +3, 지지(地支)이면 +0
+ 일간(日干)이면 +7
= 최종 ATK
```

범위: 10 (음 지지) ~ 25 (양 천간 일간)

지지 카드의 추가 속성: 지장간의 본기(本氣) 오행으로 보조 속성 부여

### 4.4 용신/기신 판별 (간이 방식)

> 현재 `SajuResult`에 용신/기신 필드가 없으므로, 기존 데이터에서 간이 판별한다.

**간이 용신 판별 알고리즘:**

```typescript
function determineYongshin(sajuResult: SajuResult): { yongshin: Element, gishin: Element } {
  const { dayMaster, fiveElements } = sajuResult;
  const dayElement = dayMaster.element; // 일간의 오행

  // 1. 신강/신약 판단 (현재 SajuResult.dayMaster.strength 활용)
  const isStrong = dayMaster.strength === '신강';

  if (isStrong) {
    // 신강 → 일간을 억제하는 오행이 용신
    // 용신: 일간을 극하는 오행 (예: 목 일간 → 금이 용신)
    // 기신: 일간을 생하는 오행 (예: 목 일간 → 수가 기신)
    return {
      yongshin: getOvercomingElement(dayElement),  // 극하는 오행
      gishin: getGeneratingElement(dayElement),     // 생하는 오행
    };
  } else {
    // 신약 → 일간을 돕는 오행이 용신
    // 용신: 일간을 생하는 오행 (예: 목 일간 → 수가 용신)
    // 기신: 일간을 극하는 오행 (예: 목 일간 → 금이 기신)
    return {
      yongshin: getGeneratingElement(dayElement),
      gishin: getOvercomingElement(dayElement),
    };
  }
}
```

**오행 상생/상극 참조:**
- 상생: 목→화→토→금→수→목
- 상극: 목→토, 토→수, 수→화, 화→금, 금→목

**주의:** 이 간이 방식은 전통 사주에서의 용신 판별(격국, 조후 등 복합 분석)과 다를 수 있으나, 게임 메카닉스 목적으로 충분하다. `dayMaster.strength`(신강/신약/중화)는 현재 SajuResult에 이미 존재한다.

|------|-----------|-----|-----|------|

### 4.6 전투 메카닉스

**방식: 1턴 카드 대결 (단순)**

전투는 복잡한 턴제가 아닌, **카드 1장 선택으로 승패가 결정**되는 간단한 구조.

```
[내 덱에서 3장 랜덤 드로 → 1장 선택]
    ↓
[오행 판정]
  - 내 카드가 몬스터를 극(克) → 승리
  - 몬스터가 내 카드를 극 → 패배
  - 같은 오행 → 내 ATK vs 몬스터 ATK 비교
  - 상생 관계 → 내 ATK ≥ 몬스터 ATK/2 이면 승리
    ↓
[승/패 결정]
```

**승리/패배 결과:**
- 승리: 보상 카드 1장 + AI가 긍정적 톤으로 풀이
- 패배: 보상 없음 + AI가 조언/주의 톤으로 풀이
- **사주 풀이 내용 자체는 동일** — 톤만 다름. 패배해도 게임 진행에 지장 없음 (다음 층으로 이동 가능)

**전투 터미널 UI:**
```
╔═══════════════════════════════╗
║  ⚔ 기신(忌神) 등장!           ║
║  "토의 장벽" (戊土)           ║
║  HP: ████████░░ 80            ║
║  속성: 토(土) | ATK: 25       ║
║  특성: 수기운(水) 억제         ║
╚═══════════════════════════════╝

현자: "그대의 수 기운을 막아서는
토의 장벽이로다..."

─── 카드를 선택하시오 ───
[1] 甲木 (갑목) ATK:20 → 상극! 효과적
[2] 壬水 (임수) ATK:15 → 억제당함
[3] 庚金 (경금) ATK:18 → 토생금 상생
```

## 5. 일일 콘텐츠 상세

### 일진(日辰) 계산

현재 `calculateFullSaju`는 생년월일 기반이므로, 일일 일진 계산을 위한 별도 유틸리티가 필요.

```typescript
// src/lib/game/daily.ts

/**
 * 오늘의 일진(日辰) 계산
 * 만세력 라이브러리의 날짜→간지 변환 기능을 활용
 * @returns { stem: string, branch: string, element: Element }
 */
```

### 오늘의 시련

- **생성:** 매일 자정, 당일 일진의 천간+지지로 몬스터 생성
- **난이도:** 내 사주와의 관계로 결정

  - 일진 오행이 일간을 생함 → 쉬움 (HP 낮음)
  - 일진이 내 사주 지지와 충(沖) → 특수 이벤트 (ATK 높음)
- **풀이:** 전투 후 AI가 오늘의 운세를 사주 관점에서 풀이
- **보상:** 해당 일진의 간지 카드 (60간지 컬렉션)

```typescript
  // 리텐션
  level: number;
  totalBattles: number;
  streak: number;
  lastVisit: string; // ISO date
  collection: string[]; // 수집한 간지 카드 ID
```

**localStorage 용량 관리:**
- AI 풀이 텍스트는 `aiCache`에 저장하되, **최대 5개 층 + 최근 일일 3일분**만 보관
- 오래된 일일 풀이는 자동 삭제
- `battleHistory`는 최근 30건만 보관
- 예상 최대 크기: ~500KB (5MB 제한 대비 여유)

**전환 규칙:**
- `card_reveal` → `floor_intro` (1층 완료 시 자동)
- `floor_intro` → `battle` (몬스터 등장 후 자동)
- `battle` → `battle_result` (카드 선택 시)
- `battle_result` → `reward` (AI 풀이 완료 시)
- `reward` → `floor_intro` (다음 층) 또는 `ending` (5층 완료 시)
- `ending` → `hub` (리포트 확인 후)
- `hub` → `floor_intro` (층 재방문) 또는 `daily` (일일 시련)

### 6.4 AI 프롬프트 확장

기존 `getRoomPrompt()` → `getFloorPrompt()` 확장:

```typescript
function getFloorPrompt(floor: number, battleResult?: BattleResult): string {
  // 기존 room prompt를 기반으로 전투 컨텍스트 추가
  const basePrompt = existingRoomPrompts[floor];

  if (battleResult) {
    const tone = battleResult.won
      ? "이 위험 요소를 극복한 관점에서 긍정적으로"
      : "이 위험 요소에 주의해야 하는 관점에서 조언하듯";

    return `${basePrompt}\n\n전투 결과: ${battleResult.won ? '승리' : '패배'}
상대: ${battleResult.monster.name} (${battleResult.monster.element})
사용 카드: ${battleResult.card.name} (${battleResult.card.element})
${tone} 풀이해주세요.`;
  }

  return basePrompt;
}
```

**엔딩 종합 프롬프트:**
```
전체 전투 기록: [2층 승, 3층 패, 4층 승, 5층 승]
전체 전투를 통해 드러난 이 사람의 사주 특성을 종합하여,
어떤 위험을 극복했고 어떤 부분을 주의해야 하는지 이야기체로 풀이해주세요.
```

## 7. 검증 방법

1. **카드 생성 검증:** 사주 입력 → 8장 기본카드가 올바른 오행/수치로 생성되는지 `npm run dev`에서 확인
2. **용신/기신 검증:** 신강/신약 사주 각각 입력하여 용신/기신이 올바르게 판별되는지 확인
3. **몬스터 생성 검증:** 각 층별 몬스터가 해당 사주의 위험 요소로 올바르게 생성되는지 확인
4. **전투 판정 검증:** 상극→승리, 역상극→패배, 같은 오행→ATK 비교 등 모든 케이스 확인
5. **AI 풀이 검증:** 승/패에 따라 풀이 톤이 달라지는지 확인 (동일 사주로 승리/패배 각각 테스트)
6. **일일 콘텐츠 검증:** 날짜별 다른 몬스터 생성, localStorage 저장/복원
7. **E2E 흐름:** 사주 입력 → 카드 공개 → 4층 전투 → 엔딩 리포트 → 허브 → 일일 시련 전체 흐름
8. **모바일 검증:** 카드 3장 나란히 표시, 전투 UI가 모바일에서 깨지지 않는지 확인
9. **localStorage 검증:** 재방문 시 데이터 로드, 스트릭 계산, 용량 초과 시 자동 정리
